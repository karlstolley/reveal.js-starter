Three Types of API

  1. Language
  2. Platform/Framework
  3. Data

JavaScript

  • The language to know, client- (browser) and server-side (Node.js)
  • Pedagogical shift for me: don’t skip/slight the language

The Course

Opening Act

Development Platform: Web Browser + DOM

Primary Concept: RTFM

First Major Project

Dummy data inserted into the DOM

Act II

Data Platform: Open Data APIs & JSONP

Primary Concept: RESTful Resources > HTTP methods

Second Major Project

Open API data inserted into the DOM

Act III, Scene 1

Development Platform: Node.js

Primary Concept: Asynchronous non-blocking I/O for DIRTy (Data-Intensive Real-Time) Applications

Act III, Scene 2

Data Platform: Protected Data APIs, XML/JSON

Primary Concepts: Authentication; Terms of service & rate-limiting

Final Project

Open/protected API data manipulated server-side (Node.js) and presented to client (Browser)

Lessons Learned

  1. Spend more time on aysnc/non-blocking I/O > “callback hell”
  2. Expand from GET to include also POST, PUT, DELETE for full stack of CRUD operations
  3. Better handling of security issues, esp. API tokens and GitHub
  4. JavaScript across the curriculum...
